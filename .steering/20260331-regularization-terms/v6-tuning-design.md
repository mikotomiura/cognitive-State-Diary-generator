# v6 チューニング設計: 検証結果に基づく改善

## 根本原因分析

7件の問題は3つの根本原因に帰着する:

### RC-1: コールドスタート問題 (P2, P4)
全ての追跡メカニズムは Day 1 で制約データが空のため、初日が完全に無制御。
Day 1 で「効率」6回、「蛍光灯」1回が使用され、以降のDayでも慣性的に反復される。
- **対策**: 累計閾値を下げ、Day 2 から制約が効くようにする + per-day 静的制限を常時注入

### RC-2: 閾値設計が保守的すぎる (P1, P2, P6)
7日間での許容回数が多すぎ、多様性確保に寄与しない。
- 余韻パターン上限3 → 7日中3日(43%)が同一パターンを許容
- 主題語ソフト15 → 前半3日で12回消費してもノーコントロール
- 古書店型は上限なし → 3/7日使用
- **対策**: 閾値を実績ベースで引き下げ

### RC-3: 検出粒度の不適合 (P3, P4, P5)
フレーズレベルの追跡は、語レベル・概念レベルの反復を捉えられない。
- prev_images: 「蛍光灯の下で...」と「蛍光灯が照らして...」は異なるフレーズだが同一マーカー
- 修辞疑問文: regex が「って、何」等の特定パターンのみで、一般的な「〜のか」を見逃す
- **対策**: マーカー語の出現日数追跡(新正規化項) + regex拡張

### RC-4: リトライ戦略の不整合 (P7)
高インパクト日では文体の破綻・短文連打が求められるが、Temperature Decayにより
保守的な生成を強制してしまい、persona_deviationが悪化する逆効果が発生。
- **対策**: 高インパクト日でpersona_deviation不合格時はtemperatureを維持

## 改善グループと変更一覧

### G1: 閾値調整 (RC-2 対策)
| 項目 | 変更前 | 変更後 | 根拠 |
|---|---|---|---|
| 余韻パターン上限 | 3回 | 2回 | 7日で2回=29%。3回=43%では多様性不足 |
| 主題語ソフトリミット | 15回 | 10回 | Day 2終了後(累計10)から制約が効く |
| 主題語ハードリミット | 25回 | 18回 | 目標15回に対し余裕を持たせつつ強制力 |
| 古書店型構造上限 | なし | 2回 | 帰路型と同等の制限 |
| per-day主題語制限 | なし | 常時3回/日 | コールドスタート問題の解消 |

### G2: 検出パターン強化 (RC-3 対策 - regex)
- `_RHETORICAL_PATTERN` に以下を追加:
  - `のだろうか[。？?]?` (問いかけ型)
  - `なのか[。？?、,]` (断定疑問型)
  - `ないのか[。？?、,]` (否定疑問型)

### G3: シーンマーカー日数追跡 (RC-3 対策 - 新正規化項)
- 新追跡変数: `scene_marker_days: dict[str, int]` (マーカー語→出現日数)
- 各Day完了後: テキスト中の_SCENE_MARKERSを検出、出現日数をインクリメント
- プロンプト注入: 3日以上出現で使用禁止、2日出現で使用自粛を要請

### G4: 高インパクト日Temperature維持 (RC-4 対策)
- `run_single_day()` のリトライループで条件判定:
  - `|emotional_impact| > 0.7` かつ 前回 `persona_deviation < 3`
  - → temperature を `max(schedule[idx], initial_temperature)` に維持
