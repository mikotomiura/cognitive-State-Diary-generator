# 依存関係確認 サブエージェント

## 目的

プロジェクトの依存パッケージ（`pyproject.toml`, `uv.lock`）を解析し、セキュリティ脆弱性、バージョン互換性、非推奨パッケージの使用、不要な依存を検出する。

---

## 検査手順

### 1. 依存関係ファイルの読み込み

- `pyproject.toml` を Read で読み込み、直接依存パッケージとそのバージョン制約を一覧化する
- `uv.lock` が存在するか確認する（存在しない場合は警告を出す）
- `[project.optional-dependencies]` の `dev` セクションの開発用依存も確認する

### 2. バージョン制約の確認

- バージョンが固定されすぎていないか（`==` で固定するとセキュリティパッチが適用できない）
- バージョンが緩すぎないか（`>=1.0` のみだと破壊的変更を含むメジャーバージョンが入る可能性）
- 推奨: `>=X.Y,<X+1` 形式（メジャーバージョン内で最新を許可）

### 3. パッケージの必要性確認

各依存パッケージが実際にコードで使用されているかを Grep で確認する。

```
pyproject.toml の各パッケージについて:
  - "import <パッケージ名>" を csdg/ 内で Grep
  - 使用箇所が0件のパッケージは不要な可能性あり
```

### 4. Python バージョン互換性

- `.python-version` のバージョンと `pyproject.toml` の `requires-python` が一致しているか
- 依存パッケージが指定した Python バージョンをサポートしているか

### 5. ロックファイルの整合性

- `uv.lock` が最新の `pyproject.toml` と同期しているか
- `uv sync` を実行した際にエラーが発生しないか

```markdown

- 🟡 更新推奨: X 件
- 🔵 情報: X 件

## 依存パッケージ一覧

### 本体依存
| パッケージ | 指定バージョン | ロック版 | 最新版 | ステータス |

| pydantic | >=2.0 | 2.6.1 | 2.7.0 | 🟡 更新可能 |
| openai | >=1.0 | 1.12.0 | 1.12.0 | ✅ 最新 |

| パッケージ | 指定バージョン | ロック版 | 最新版 | ステータス |
|---|---|---|---|---|

#### [D-01] pydantic 2.7.0 が利用可能

- **最新:** 2.7.0
- **変更内容:** バグ修正、パフォーマンス改善

- **パッケージ:** (パッケージ名)
- **理由:** csdg/ 内で import が見つからない
- **推奨:** 使用していない場合は依存から削除する

## ロックファイル状態

## 推奨アクション
1. (優先度の高いアクションから順に)
```

---

## 注意事項

- 脆弱性の判定はパッケージのバージョン情報とリリースノートに基づいて行う
- メジャーバージョンアップの推奨は慎重に行い、破壊的変更の有無を確認する
- 不要パッケージの判定では、間接的な使用（pytest のプラグイン等）も考慮する
