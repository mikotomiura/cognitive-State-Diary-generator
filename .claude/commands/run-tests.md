# /run-tests — テスト実行・分析ワークフロー

> **目的:** テストスイートを実行し、結果を分析し、問題があれば改善アクションを提示する。
> **粒度:** 1回のテストセッション。全テスト実行 or 指定モジュールのテスト。

---

## ステップ 1: テスト対象の決定

以下のいずれかを選択する:

- **全テスト実行:** 変更範囲が広い場合、コミット前の最終確認
- **モジュール指定:** 特定のモジュールのみを変更した場合
- **失敗テストのみ再実行:** 直前の実行で失敗したテストの修正確認

---

## ステップ 2: テストの実行

1. **`test-runner` サブエージェントを起動する**

   指定に応じて以下のいずれかを実行させる:

   ```bash
   # 全テスト（カバレッジ付き）
   pytest tests/ -v --cov=csdg --cov-report=term-missing -m "not e2e"

   # モジュール指定
   pytest tests/test_schemas.py -v --cov=csdg/schemas.py --cov-report=term-missing

   # 失敗テストのみ再実行
   pytest tests/ -v --lf --tb=long
   ```

2. **実行結果レポートを受け取る**

---

## ステップ 3: 結果の判定

### 全テスト Pass の場合

1. カバレッジが目標値を満たしているか確認する:

   | モジュール | 目標 |
   |---|---|
   | `schemas.py` | 95% |
   | `config.py` | 90% |
   | `engine/critic.py` | 95% |
   | `engine/actor.py` | 80% |
   | `engine/pipeline.py` | 85% |
   | `scenario.py` | 90% |
   | `visualization.py` | 70% |

2. 目標未達の場合、カバレッジされていない行を確認し、追加テストの方針を決定する

### テスト失敗がある場合

1. **`test-analyzer` サブエージェントを起動する**
2. 根本原因分析レポートを受け取る
3. 原因に応じて対応を決定する:
   - **テストの誤り:** テストを修正する
   - **実装のバグ:** `/fix-bug` ワークフローに移行する
   - **スキーマ変更の反映漏れ:** テストとコードの整合性を修正する

---

## ステップ 4: 型チェック・リンターの併走

テスト結果と合わせて、型チェックとリンターも実行する:

```bash
mypy csdg/ --strict
ruff check csdg/
```

```markdown
- リンター: ✅ / ❌

- (具体的なアクション)
```
